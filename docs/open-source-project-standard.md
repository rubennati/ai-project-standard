# Open Source Project Standard

This project defines a practical minimum baseline for a professional repository.

## Baseline components

- `README.md` for scope and orientation
- `LICENSE` for legal clarity
- `CONTRIBUTING.md` for contribution workflow
- `SECURITY.md` for vulnerability handling
- issue and pull request templates for structured collaboration
- GitHub Actions for basic automated checks
- `docs/` for stable process documentation

## Governance requirements

- branch protection on `main`
- pull request review before merge
- clear directory responsibilities
- maintainability-focused change control

## AI collaboration readiness

The repository should expose explicit routing, state, decisions, and quality constraints so AI-assisted output remains auditable and controllable by humans.
