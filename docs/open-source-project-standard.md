# Open Source Project Standard

This project defines a minimum professional baseline for modern repositories.

## Baseline components

- `README.md` for scope and orientation
- `LICENSE` for legal clarity
- `CONTRIBUTING.md` for contribution workflow
- `SECURITY.md` for vulnerability reporting
- issue templates for structured intake
- GitHub Actions for automated checks
- documentation in `docs/`

## Governance expectations

- branch protection on `main`
- pull request review before merge
- clear repository structure
- maintainability-focused change management

## AI collaboration readiness

A repository should expose explicit rules, state, routing, and quality criteria so AI-generated work remains reviewable and controllable by humans.
