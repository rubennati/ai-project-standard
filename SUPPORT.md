# Support

Where to ask, where to file, what to expect.

## Issues are for actionable work

Open a [GitHub Issue](https://github.com/rubennati/ai-project-standard/issues/new/choose) when you have a concrete, actionable item:

- A **bug** with reproduction steps.
- A specific **feature request** with a clear motivation.
- A **documentation gap** with a missing or incorrect statement to fix.

Issues that are actually open-ended questions, "is anyone using this?" threads, or general feedback will be converted to Discussions.

## Discussions are for everything else

Use [GitHub Discussions](https://github.com/rubennati/ai-project-standard/discussions) for:

- **Questions** about how to adopt the standard.
- **Ideas** that are not yet concrete enough to be issues.
- **Show & Tell** — projects that have adopted the standard.
- **Announcements** from maintainers.

If a discussion converges on a specific change to the standard, a maintainer will spin off an issue from it.

## Security issues

**Do not open public issues for security concerns.** Follow [SECURITY.md](./SECURITY.md) and report privately via GitHub Security Advisories.

## Response expectations

This is a solo-maintained, evenings-and-weekends project. Realistic expectations:

| Channel | Target first response |
|---|---|
| Security advisory | within a few days |
| Bug with reproduction | within a week |
| Feature request | within two weeks |
| Discussion | best effort |

Maintainers will not always meet these targets, especially during release cycles. If something is urgent, say so in the title.

## External support channels

There are no external support channels for this project. Everything lives in this repository.
